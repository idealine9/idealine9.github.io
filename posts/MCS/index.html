6. State Machine

A state machine is a binary relation on a set, the elements of the set are called 'states', the relation is called the transition relation, an arrow in the graph of the transition relation is called a transition. The transition relation is called the state graph of the machine. A state machine also comes equipped with a designated start state.

6.2 Invariant Principle

A state machine execution describes a possible sequence of steps a machine might take.

Definition 6.2.4

An execution of thee state machine is a (possible infinite) sequence of states with the propperty that

  • it begins with the start state
  • if \(q\) and \(r\) are consecutive states in the sequences, then \(q \rightarrow r\) .

A state is called reachable if it appears in some execution.

Definition 6.2.5

A preserved invariant of a state machine is a predicate \(P\) on states, such that whenever \(P(q)\) is true of a state \(q\) and \(q \rightarrow r\) for state \(r\), then \(P(r)\) holds.

The Invariant Principle

If a preserved invariant of a state machine is true for the start state, then it is true for all reachable states.

10. Group theory

Directed graph

10.1 Vertex Degrees

The \(in {-} degree \ (d^-)\) is the number of arrows coming into it ( number of times the vertex is pointed ),

similarly, the \(out{-}degree \ (d^+)\) is the number of arrows out of it ( number of points to other vertices ).

If \(G\) is a digraph and \(v \in V(G)\) , then \[ \sum_{v \ \in \ V(G)} d^+(v) = \sum_{v \ \in \ V(G)} d^-(v) \] and both sums are equal to \(\vert E(G) \vert\)

10.2 Walks and Paths

Definition 10.2.1

\(walk\) : A \(walk\) in a digraph is an alternating sequence of vertices and edges that begins with a vertex and ends with a vertex.

\(closed \ walk\) : A \(closed \ walk\) is a \(walk\) that begins and ends at the same vertex.

\(path\) : The \(walk\) is a \(path\) iff all the vertices are different.

\(cycle\) : A \(cycle\) is a positive length \(path\) and begins and ends at the same vertex.

Lemma 10.2.5

The Triangle Inequality

\[ dist(u,v) \le dist(u,x)+dist(x,v) \]

for all vertices \(u,\ v, \ x\) with equality holding if and only if x is on a shortest path from \(u\) to \(v\).

10.3 Adjacency Matrices

If a graph \(G\) has \(n\) vertices \(v_0, \ v_1 \ ... \ v_{n-1}\) , a useful way to represent it is with an \(n \times n\) matrix of zeros and ones called its adjacency matrices \(A_G\).

\((A_G)_{ij}\) is \(1\) if there is an edge from vertex \(v_i\) to vertex \(v_j\) and \(0\) otherwise. \[ (A_G)_{ij}= \begin{cases} 1 & if \ \langle v_i \to v_j \rangle \in E(G), \\ 0 & otherwise, \end{cases} \]

Let \(H\) be the 4-node graph. Its adjacency matrix \(A_H\) is the 4 \(\times\) 4 matrix : \[ A_H \ = \ \ \ \begin{array}{} & a & b & c & d \\ a & 0 & 1 & 0 & 1 \\ b & 0 & 0 & 1 & 1 \\ c & 0 & 1 & 0 & 0 \\ d & 0 & 0 & 1 & 0 \\ \end{array} \] We can use matrix powers to count numbers of walks between vertices. For example, if we want to know the walk with a length of \(2\), we could have read these counts from the entires in the matrix \((A_H)^2\) : \[ (A_H)^2 \ = \ \ \ \begin{array}{} & a & b & c & d \\ a & 0 & 0 & 2 & 1 \\ b & 0 & 1 & 1 & 0 \\ c & 0 & 0 & 1 & 1 \\ d & 0 & 1 & 0 & 0 \\ \end{array} \] More generally, the matrix \((A_G)^k\) provides a count of the number of length \(k\) walks between vertices in any digraph \(G\).

Theorem 10.3.2

For a digraph \(G\), if \(C\) is the length-m walk counting matrix and \(D\) is the length-n walk counting matrix, then \(CD\) is the length k+m walk counting matrix for \(G\).

Corollary 10.3.3

The length-k counting matrix of a digraph \(G\) is \((A_G)^k\) , for all \(k\in\) \(\mathbb{N}\).

In other words, you can determine the number of length \(k\) walks between any pair of vertices simply by computing the \(k\) th power of the adjacency matrix.

Shortest Path

To find the lengths of all the shortest paths in an n-vertex graph \(G\) is to compute the successive powers of \(A_G\) one by one up to the \(n-1\) times, watching for the first power at which each entry becomes positive.

The distance from \(u\) to \(v\) will be the smallest value \(k\) for which \((A_G)^K_{uv}\) is nonzero, and if there is a shortest path, its length will be \(\le n-1\) . The methods apply as well to weighted graphs.

10.4 Walks Relations

For any digraph \(G\), a binary relation \(G^*\), called the walk relation on \(V(G)\), where \[ u \ G^* \ v ::= \text{there is a walk in $G$ from $u$ to $v$} \]

10.5 Directed Acyclic Graphs(DAG) & Scheduling

Definition 10.5.1

A directed acyclic graph (DAG) is a directed graph with no cycles.

Definition 10.5.2

A topological sort of a finite DAG is a list of all the vertices such that each vertex \(v\) appears earlier in the list than every other vertex reachable form \(v\).

Definition 10.5.3

  1. A vertex \(v\) of a DAG \(G\) is minimum if and only if every other vertex is reachable from \(v\).
  2. A vertex \(v\) of a DAG \(G\) is minimal if and only if \(v\) is not reachable from any other vertex.

Definition 10.5.4

Every finite DAG has a topological sort.

Definition 10.5.5

Two vercices in a DAG are comparable when one of them is reachable from the other.

A chain in a DAG is a set of vertices such that any two of them are comparable.

A vertex in a chain that is reachable from all other vertices in the chain is called a maximum element of the chain. A finite chain ends at its maximum element.

A largest chain is also known as a critical path.

Definition 10.5.6

A partition of a set \(A\) is a set of nonempty subsets of \(A\) called the blocks of the partition, such that every element of \(A\) is in exactly one block.

Definition 10.5.7

A parallel schedule for a DAG \(D\) is a partition of \(V(D)\) into blocks \(A_0, A_1,...\) , such that when \(j<k\) , no vertex in \(A_j\) is reachable from any vertex in \(A_k\)

For example, one possible partition of the set \(\{ a,b,c,d,e \}\) into three blocks is \[ \{a,c\} \ \ \ \{b,e\} \ \ \ \{d\} \]

The block \(A_k\) is called the set of elements scheduled at step k , and the time of the schedule is the number of blocks. The maximum number of elements scheduled at any step is called the number of processors required by the schedule.

A largest chain ending at an element \(a\) is called a critical path to \(a\), and the number of elements less than \(a\) in the chain is called the depth of \(a\).

So, in any possible parallel schedule, there must be at least \(depth(a)\) steps before task \(a\) can be started.

In particular, the minimal elements are precisely the elements with depth 0.

12.4 Graph isomorphism

Definition 12.4.1

An isomorphism between graphs \(G\) and \(H\) is a bijection \(f : V(G) \rightarrow V(H)\) such that \[ \langle u - v \rangle \in E(G) \quad iff \quad \langle f(u)-f(v) \rangle \in E(H) \]

for all \(u,v \in V(G).\) Two graphs are isomorphism when there is an isomorphism between them.

Isomorphism preserves the connection properties of a graph, abstracting out what the vertices are called, what they are made out of, or where they appear in the graph. More precisely, a property of a graph is said to be preserved under isomorphism if whenever \(G\) has that property, every graph isomorphic to \(G\) also has that property.

If \(f\) is a graph isomorphis, that maps a vertex \(v\) of one graph to the vertex \(f(v)\) of an isomorphic graph, then by definition of isomorphism, every vertex adjacent to \(v\) in the first graph will be mapped by \(f\) to a vertex adjacent to \(f(v)\) in the isomorphic graph. Thus, \(v\) and \(f(v)\) will have the same degree.

12.5 Bipartite Graph & Matching

Definition 12.5.1

A bipartite graph is a graph whose vertices can be partitioned into two blocks, \(L(G)\) and \(R(G)\) , such that every edge has one endpoint in \(L(G)\) and the other endpoint in \(R(G)\).

Matching

Definition 12.5.3

A matching in a graph is a set of edges \(M \in E(G)\) such that no two edges in \(M\) are incident to the same vertex. The vertices incident to an edge in \(M\) are said to be coverd by \(M\). A matching covers a set of vertices \(S\), each vertex in \(S\) is a vertex of a matching edge. A matching is said to be perfect when it covers \(V(G)\). The set \(N(S)\) of neighbors of some set \(S \in V(G)\) of vertices is the image of \(S\) under the edge-relation, that is

\[ N(S) ::= \{ r \ | \ \langle s-r \rangle \in E(G) \ \text{for some} \ s \in S\} \]

\(S\) is called a bottleneck if \[ \lvert S \rvert > \lvert N(S) \rvert \]

Theorem 12.5.4

(Hall's Theorem) Let \(G\) be a bipartite graph. There is a matching in \(G\) that covers \(L(G)\) iff no subset of \(L(G)\) is a bottleneck.

12.6 Coloring

Given a graph \(G\) , assign colors to each node such that adjacent nodes have different colors. A color assignment with this property is called a valid coloring of the graph-a "coloring", A graph \(G\) is \(k\)-colorable if it has a coloring that uses at most \(k\) colors.

Definition 12.6.1

The minimum number of colors for which a graph \(G\) has a valid coloring is called its chromatic number, \(\chi(G)\).

\(G\) is \(k\)-colorable iff \(\chi(G) \leq k\).

An even-length cycle is 2-colorable: \[ \chi(C_{\text{even}}) = 2 \] An odd-length cycle is 3-colorable: \[ \chi(C_{\text{odd}}) = 3 \]

A complete graph \(K_n\) : \[ \chi(K_n) = n \] since no two vertices can have the same color.

Lemma 12.6.2

A nonempty graph \(G\) is bipartite iff \(\chi(G) = 2\).

Definition 12.6.3

A graph with maximum degree at most \(n\) is \((n+1)\)-colorable.

12.7 Walks in Simple Graph

We don't intend that cycles have a beginning or an end, so any of the paths that go around a cycle can represent it. Furthermore, cycles in simple graphs don't have a direction.

A precise way to explain which closed walks represent the same cycle is to define a cycle to be subgraph isomorphic to a cycle graph \(C_n\).

Definition 12.7.2

A graph \(G\) is said to be a subgraph of a graph \(H\) if \(V(G) \subseteq V(H)\) and \(E(G) \subseteq E(H)\).

12.8 Connectivity

Definition 12.8.1

Two vertices are connected in a graph when there is a path that begins at one and ends at the other. A graph is connected when every pair of vertices are connected.

Connected Components

Definition 12.8.2

A connected components of a graph is a subgraph consisting of some vertex and every node and edge that is connected to that vertex.

So a graph is connected iff it has exactly \(1\) connected component. At the other extreme, the empty graph on \(n\) vertices has \(n\) connected components, each connected component consisting of a single vertex.

Odd cycles and 2-Colorability

Theorem 12.8.3

The following graph properties are equivalent: 1. The graph contains an odd length cycle. 2. The graph is not 2-colorable. 3. The graph contains an odd length closed walk.

In other words, if a graph has any one of the three properties above, then it has all of the properties.

12.9 Eular Tours & Hamiltonian Cycles

An Euler tour of a graph is a closed walk that includes every edge exactly once.

Theorem 12.9.1

A connected graph has an Euler tour iff every vertex has even degree.

Hamiltonian cycles are the unruly cousins of Euler tours.

Definition 12.9.2

A hamiltonian cycle in a graph \(G\) is a cycle that visits every vertex in \(G\) exactly once. Similarly, a hamiltonian path is a path in \(G\) that visits every vertex exactly once.

12.10 k-connected Graphs

Definition 12.8.4

Two vertices in a graph are \(k\)-edge connected when they remain connected in every subgraph obtained by deleting up to \(k-1\) edges. A graph with two or more vertices is \(k\)-edge connected when every pair of distinct vertices in the graph are \(k\)-edge connected.

Two vertices are \(k\)-edge connected when it takes at least \(k\) "edge-failures" to disconnect them.
